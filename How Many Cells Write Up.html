def croptwoclusts(X, y, clust1, clust2):
    clust1 = clustindex(clust1)
    clust2 = clustindex(clust2)
    mask = np.logical_or(y == clust1, y == clust2)
    Xm, ym = X[mask], y[mask]
    return Xm, ym, mask

def computeseparability(X, y, k=20):
    neighs = scipydist.squareform(scipydist.pdist(X)).argsort(axis=1)
    neighy = y[neighs[:,1:k+1]]
    correctneigh = np.equal(neighy, y.reshape(-1, 1))
    return correctneigh.sum()/correctneigh.size

def computesil1(X, y, clust1, clust2):
    clust1, clust2 = clustindex(clust1), clustindex(clust2)
    X, y, _ = croptwoclusts(X, y, clust1, clust2)
    x1bar = X[y==clust1].mean(axis=0)
    x2bar = X[y==clust2].mean(axis=0)
    d = x1bar - x2bar
    d = d.reshape((-1, 1))
    Xproj = X @ d
    return silhouette_score(Xproj, y)

Projected Silhouette Score¶

One measure of cluster overlap is silhouette score, defined as $\frac{a - b}{\max(a,b)}$, where $a$ is the mean distance to points in the same cluster and $b$ is the distance to points in the nearest cluster (in our case, we only look at two clusters). Note that a negative score suggests points across clusters are closer than points within clusters, a zero score suggests the clusters overlap, and a score of 1 suggests the clusters are completely distinct.

As stated, silhouette score would either be computed in very high dimension (bad!) or in PC space (whose selection is affected by other clusters).

We propose the following way of measuring overlap that does not use PCA. If $\bar x_i$ and $\bar x_j$ are the centroids of clusters, we project the dataset along $\vec d = \bar x_i - \bar x_j$ and then compute the silhouette score. That is, if $X$ is a n_cell x n_genes data matrix, we compute the silhouette score on $X \vec d$ (note that silhouette score is invariant to scaling and translation, so we needn't be careful about how we project).

Robustness to n_cell¶

If we are going to measure cluster overlap in a meaningful way, we need it to be robust to the number of cells we've downsampled to, so that this is a meaningful measure when we only have a small dataset with which we hope to design a larger experiment.

Other Factors¶

Our model is a quasibinomial GLM--it's nonlinear and allows for overdispertion. We use the following factors:

  • logit((projected_silhouette_score + 1)/2)
  • log(n_cell), where n_cell is the number of cells in the sample
  • log(freq1 * freq2), where freq1 and freq2 are the frequencies of the two clusters

Future Considerations: Asymptotic Separability¶

Our model never predicts an asymptotic separability score of $< 1$. We considered modeling asymptotic score as a function of the projected silhouette score, but this didn't work well on the retina dataset. We suspect this is because many clusters here arise from demarcated trajectories without any buffer between them. In this case, the projected silhouette score is low (because distance between clusters is small, even though there isn't a lot of overlap). This demonstrates the need for a better score for overlap so that we can scale our prediction based on an estimated asymptotic separability.
